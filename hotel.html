Hotel Booking System

In this project, an 8x8 array is initialised to represent the rooms of a hotel. Commands can then be input by the user to interact with the hotel.

Through this project I was introduced to array manipulation (e.g. adding and removing bookings to specific rooms) and variable status checking (e.g. before adding a new booking to a room, check whether the room is already occupied). Edge cases were a big challenge in this task since many of the individual functions interact with and rely on other factors to produce the correct output.

The full list of commands is listed here. Note that a '+' in the commands column represents a follow up input. Also [x][y] coordinates require a space between the two values. For example, '? + [x][y]' means to input '?' and then input '[x] [y]'.

Command Usage
p Print the hotel into the terminal.
? + [x][y] Check if room [x][y] is occupied.
a + [x][y] Add a booking to room [x][y].
l + [x][y] Remove a booking from room [x][y].
c Print the hotel with prices.
$ + [n][p] Change the price of all rooms on floor [n] to price [p].
^ + [p] Change the price of all rooms in the hotel to price [p].
r Calculate the revenue earned. Adds the prices of occupied rooms together and compares the total to the previous revenue output.
j Finds the next best room and adds a guest. A 'best room' is defined by the leftmost and uppermost room available.
s + [x1][y1][x2][y2] Swaps the booking characteristics of rooms [x1][y1] and [x2][y2].
g + 1 + [n] Add a group booking of [n] rooms to the cheapest and lowest floor.
g + 2 + [n][p] Add a group booking of [n] rooms within a budget of [p].
! + [x][y] Add a royal guest to the hotel. A royal guest gets priority over their desired room [x][y], moving any occupants to the next available room.
